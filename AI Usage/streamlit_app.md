# AI Usage Documentation (Streamlit)

## Overview

Artificial intelligence tools were used in this project to support the transformation of exploratory analysis into a structured and presentation-ready Streamlit application. 
The core data analysis, statistical reasoning, and interpretation of results were performed independently. AI was primarily used to improve code organization, visualization clarity, 
and user interface structure.

---

## Scope of AI Usage

AI assistance was used in the following areas:

- Structuring the Streamlit application layout  
- Converting notebook-based analysis into dashboard components  
- Improving visualization formatting and readability  
- Debugging deployment issues (e.g., file paths, dependencies)  

AI was not used to generate the research questions, define the methodology, or interpret the final results.

---

## Example Prompts Used

### 1. Converting Analysis into Streamlit Components

Prompt:
"Convert this seaborn regression plot into a Streamlit-compatible visualization and structure it as a section in a dashboard."

Purpose:
To translate notebook-based visualizations into an interactive dashboard format.

---

### 2. Improving Visualization Presentation

Prompt:
"Improve this seaborn bar chart to make it more presentation-ready with better styling, labels, and readability."

Purpose:
To enhance clarity and visual quality of charts for presentation purposes.

---

### 3. Structuring the Dashboard

Prompt:
"Create a clean Streamlit app structure with sections for pricing analysis, customer behavior, sales trends, and crypto volatility."

Purpose:
To organize the project into logical sections aligned with the research questions.

---

### 4. Debugging Deployment Issues

Prompt:
"Why is my Streamlit app failing with ModuleNotFoundError and FileNotFoundError when deployed?"

Purpose:
To resolve environment-related issues such as missing dependencies and incorrect file paths.

---

### 5. Handling Large Datasets

Prompt:
"How can I load large datasets efficiently in Streamlit Cloud?"

Purpose:
To implement solutions such as data compression (.csv.gz) and caching for performance optimization.

---

### 6. Map Visualization

Prompt:
"Create a global map visualization showing revenue by country grouped into tiers (low, medium, high)."

Purpose:
To implement a choropleth map using Plotly for geographic revenue analysis.

---

## Human Contribution

The following aspects of the project were completed without AI generation:

- Formulation of business questions and hypotheses  
- Data cleaning and preprocessing  
- Statistical analysis and interpretation  
- Insight generation and business conclusions  
- Design of analytical approach  

AI outputs were reviewed, modified, and integrated to ensure correctness and alignment with the project objectives.

---

## Limitations of AI Usage

- AI-generated code required validation and adjustment to match the dataset structure  
- Some outputs required debugging due to environment differences  
- Interpretations were not taken directly from AI and were independently verified  

---

## Conclusion

- AI was used as a supportive tool to enhance productivity and presentation quality. 
- All critical thinking, analysis, and conclusions were developed independently.
- AI serving only as an aid in structuring and refining the final deliverables.
